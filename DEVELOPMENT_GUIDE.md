# LUMEN BANK - Development Guide & Architecture

## 1. Общая концепция
LUMEN — это высокотехнологичное банковское PWA. Основной акцент сделан на UX/UI (Apple-style), безопасности (PIN, KYC, AML) и поддержке мультикуррентности (Fiat, Crypto, Smart Contracts).

## 2. Рекомендуемая структура проекта
Для масштабирования проекта следует перейти от плоской структуры к модульной:

```text
src/
├── api/              # Инстансы axios/fetch, сервисы для разных сущностей
│   ├── user.js       # /api/user/*
│   ├── cards.js      # /api/cards/*
│   └── transactions.js
├── assets/           # Иконки, изображения, глобальные стили
├── components/       # Общие UI-компоненты (Buttons, Inputs, Modals)
│   └── ui/           # Атомарные компоненты (BaseButton.jsx и т.д.)
├── context/          # Глобальное состояние (AppContext.jsx)
├── hooks/            # Кастомные хуки (useAuth, useBalance, useKyc)
├── i18n/             # Локализация (en.json, fr.json)
├── screens/          # Страницы приложения (Разделение по модулям)
│   ├── auth/         # Onboarding, PinLogin
│   ├── main/         # Home, History, Profile
│   ├── finance/       # Cards, Transfers, Credit
│   └── crypto/       # Crypto, SmartContracts
└── utils/            # Хелперы (formatCurrency, formatDate)
```

## 3. Правильная логика данных (Data Flow)

### Уровни ответственности:
1.  **API Layer (`src/api/`)**: Только выполнение HTTP-запросов. Не знает о стейте приложения, только возвращает данные или ошибки.
2.  **Context/State Layer (`src/context/`)**: Хранит текущее состояние пользователя, сессию и кэш данных. Оркестрирует вызовы API и обновляет глобальный стейт.
3.  **Hook Layer (`src/hooks/`)**: Прослойка для конкретных функций (например, `useTransaction` может содержать логику валидации суммы перед отправкой в API).
4.  **View Layer (`src/screens/`)**: Только отображение данных и отправка событий в хуки/контекст.

## 4. Дорожная карта развития (Logic Roadmap)

### Этап 1: Безопасность и Аутентификация
- [ ] **JWT Auth**: Заменить хранение PIN в `localStorage` на полноценные JWT-токены (AccessToken/RefreshToken).
- [ ] **Biometrics API**: Интеграция WebAuthn для FaceID/TouchID.
- [ ] **Session Timeout**: Автоматический выход из системы при неактивности.

### Этап 2: Финансовый модуль
- [ ] **Валидация транзакций**: Проверка баланса на стороне бэкенда перед подтверждением перевода.
- [ ] **Dynamic Cards**: Возможность создания виртуальных карт через приложение (запрос -> одобрение админом).
- [ ] **Smart Contracts**: Реализация логики «блоков» (выплата при выполнении условий).

### Этап 3: Администрирование (AdminApp)
- [ ] **User Approval**: Система верификации заявок с онбординга (Pending -> Active).
- [ ] **Money Management**: Инструменты для начисления баланса пользователям.
- [ ] **KYC/AML Control**: Ручное управление статусами проверки документов.

## 5. Стандарты кода
- **Стили**: Tailwind CSS (обязательно использовать конфигурацию `tailwind.config.js` для брендовых цветов `lumen-black`).
- **Анимации**: Framer Motion для всех переходов между экранами.
- **Типизация**: Рекомендуется перевод на TypeScript для исключения ошибок в финансовых расчетах.
- **I18n**: Все текстовые строки должны проходить через функцию `t(key)`.
