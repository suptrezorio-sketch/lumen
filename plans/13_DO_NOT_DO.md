# 13 — Do Not Do

## Product mistakes

Do not:

```text
turn Lumen into a toy demo
show training labels inside user flow
make balances plain UI text
use random card numbers not linked to backend entities
use random wallet addresses not linked to backend entities
hide frequent actions in deep menus
replace Home with generic dashboard
add too many bottom tabs
```

## Admin mistakes

Do not build an admin like:

```text
one tab row
three user cards
green check / red X
balance input
giant empty white page
```

Admin must be:

```text
Client Card
Pending Queue
KYC Review
Document Review
Operations
Ledger
Smart Contracts
Scenario Templates
Audit
```

## Design mistakes

Do not:

```text
mix random design systems
ignore Framework7
use tiny input fonts on mobile
use manual date text fields
use browser keyboard everywhere in PWA
use sticky backdrop blur on iOS PWA
use spring animations on toggles
use colors without purpose
copy old AML HTML visual style
```

## Language mistakes

Do not use:

```text
Greek language
trigger words
fake
demo
simulator
scam
```

Use:

```text
Wallet Risk Training
Wallet Verification
Smart Contract
Settlement Order
Processing
Pending
Review
```

## Data mistakes

Do not:

```text
store sensitive scenario answers in plain text
send sensitive values to analytics
log documents into console
track card numbers
track passport values
track training_recovery_phrase
let clients update balances directly
let admins change data without audit log
```
